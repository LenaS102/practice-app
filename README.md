ИНСТРУКЦИЯ ПО ЗАПУСКУ ПРИЛОЖЕНИЯ
Наименование программы: Обработка изображений – Вариант 20

Назначение программы: Десктоп-приложение для обработки изображений, разработанное в рамках ознакомительной практики. Программа позволяет загружать изображения в форматах PNG и JPG, делать снимки с веб-камеры, просматривать цветовые каналы (красный, зелёный, синий), а также выполнять три функции по варианту 20: обрезку изображения, изменение яркости и рисование зелёной линии.

Студент: Соколова Е.А.

Группа: ЗКИ25-18Б

Вариант: 20

1. Требования к системе
Для запуска приложения необходимо:

Операционная система: Windows, macOS или Linux.

Python версии 3.6 или выше.

Веб-камера (необязательно, только для функции съёмки).

Подключение к интернету (для установки библиотек).

2. Состав приложения
Приложение включает следующие файлы:

main.py – основной код программы.

requirements.txt – список библиотек, необходимых для работы.

README.md – инструкция по установке и запуску.

3. Установка и запуск
3.1. Получение кода
   
Скачайте архив с кодом из репозитория:

https://github.com/LenaS102/practice-app

На странице репозитория нажмите кнопку Code, выберите Download ZIP и распакуйте архив в удобную папку.

Либо выполните в терминале команду:

git clone https://github.com/LenaS102/practice-app.git
cd practice-app

3.2. Создание виртуального окружения
Откройте командную строку или терминал в папке с проектом и выполните:

python -m venv venv

После выполнения в папке появится папка venv.

3.3. Активация виртуального окружения
Для Windows (PowerShell):

venv\Scripts\Activate.ps1

Для Windows (CMD):

venv\Scripts\activate.bat

Для macOS и Linux:

source venv/bin/activate

После активации в начале строки появится (venv).

3.4. Установка зависимостей
Выполните команду:

pip install -r requirements.txt

Будут установлены библиотеки opencv-python, numpy и Pillow, указанные в файле requirements.txt. Библиотека Tkinter входит в состав Python и не требует отдельной установки.

3.5. Запуск приложения
Выполните команду:

python main.py

Откроется графическое окно программы.

4. Работа с приложением
Программа содержит семь кнопок управления:

Загрузить изображение – выбор файла PNG или JPG.

Снять с камеры – запуск веб-камеры и создание снимка.

Цветовые каналы – просмотр красного, зелёного или синего канала.

Обрезка – ввод координат X1, Y1, X2, Y2 для обрезки изображения.

Яркость – ввод значения от -100 до 100 (положительное – светлее, отрицательное – темнее).

Зелёная линия – ввод координат начала и конца линии, а также толщины.

Сброс – возврат к исходному изображению.

5. Возможные проблемы и их решение

При установке и запуске приложения на разных операционных системах
могут возникнуть следующие типичные проблемы.

5.1. Ошибка при запуске программы: "ModuleNotFoundError: No module named 'cv2'"
Библиотека OpenCV не установлена или установлена вне виртуального окружения.

Решение для Windows (PowerShell):
venv\Scripts\activate
pip install -r requirements.txt

Решение для Windows (CMD):
venv\Scripts\activate.bat
pip install -r requirements.txt

Решение для macOS и Linux:
source venv/bin/activate
pip install -r requirements.txt

При необходимости установить библиотеку отдельно:
pip install opencv-python

5.2. Ошибка при попытке загрузить изображение с русскими буквами в названии файла или пути к нему.
В программе используется функция cv2.imdecode с np.fromfile, поддерживающая кириллицу. Однако если ошибка всё же возникает, рекомендуется сохранить изображение в папку с латинскими буквами в названии (например, C:\images\) или переименовать файл, используя только латинские буквы.

Решение одинаково для всех операционных систем.

5.3. Веб-камера не работает или не открывается.
Возможные причины и решения в зависимости от операционной системы:

Для Windows:
- Проверить, что камера подключена и не используется другим приложением.
- Перезагрузить компьютер или установить/обновить драйверы для веб-камеры
  с сайта производителя.

Для macOS:
- При первом запуске программы система может запросить разрешение на доступ к камере. Если разрешение не было выдано, откройте:
  "Системные настройки" → "Безопасность и конфиденциальность" → "Конфиденциальность" → "Камера" и разрешите доступ для терминала (или приложения Python).
- После выдачи разрешения перезапустите программу.

Для Linux:
- Убедитесь, что камера определена системой (команда: ls /dev/video*).
- Если устройство не отображается, установите пакет v4l-utils:
  sudo apt install v4l-utils (для Ubuntu/Debian)
  sudo dnf install v4l-utils (для Fedora)
  После этого требуется перезагрузка или перелогин.
- Проверьте работу камеры командой: cheese (если установлена).

Общее для всех ОС:
- Убедиться, что камера не используется другим приложением.
- Перезагрузить компьютер.

5.4. Ошибка при активации виртуального окружения в Windows PowerShell:

Решение для Windows (PowerShell):
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Затем нажать A (Да для всех) и повторить активацию окружения.

Решение для Windows (CMD):
Данная ошибка не возникает в CMD.

Решение для macOS и Linux:
Данная ошибка не возникает, так как в этих системах используется
команда source venv/bin/activate.

5.5. При запуске программы открывается чёрное окно консоли, а не графическое приложение.
Решение: программа предназначена для запуска через командную строку.
Откройте командную строку в папке с проектом, активируйте виртуальное окружение и выполните команду python main.py.

Решение одинаково для всех операционных систем.

5.6. Ошибка: pip не является внутренней или внешней командой.
Решение: обновить pip командой:
python -m pip install --upgrade pip
После обновления повторить установку зависимостей.

Решение одинаково для всех операционных систем.

5.7. При загрузке изображения программа не реагирует или закрывается.
Решение: убедиться, что файл имеет поддерживаемый формат (PNG или JPG) и не повреждён. Попробовать открыть файл в стандартном просмотрщике изображений.

Решение одинаково для всех операционных систем.

5.8. Не удаётся найти файл requirements.txt при установке зависимостей.

Решение: убедиться, что командная строка открыта в папке с проектом, где находится файл requirements.txt.

Для этого выполните команду проверки:
- Windows: dir requirements.txt
- macOS и Linux: ls requirements.txt

Если файл не найден, перейдите в папку с проектом с помощью команды cd.
Например, если проект распакован на рабочий стол:
- Windows: cd C:\Users\Имя_пользователя\Desktop\practice-app-main
- macOS и Linux: cd ~/Desktop/practice-app-main

Рекомендация: используйте папки с английскими названиями, чтобы избежать проблем с путями на разных операционных системах.

5.9. Ошибка при запуске программы: "python: command not found"

На некоторых операционных системах (macOS, Linux) команда python может отсутствовать или быть привязана к устаревшей версии Python 2.

Для macOS и Linux:
Использовать команду python3 вместо python.
python3 main.py

Для Windows:
Команда python main.py работает стандартно.

5.10. Ошибка при установке зависимостей на Linux: отсутствие прав доступа

При установке пакетов через pip может возникнуть ошибка, связанная с недостаточными правами доступа к системным папкам.

Для Linux:
Не использовать sudo для установки пакетов. Создать виртуальное окружение с правами пользователя и установить зависимости внутри него:
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

Для Windows:
Обычно проблем с правами доступа не возникает. Если ошибка появляется, запустить командную строку от имени администратора.
